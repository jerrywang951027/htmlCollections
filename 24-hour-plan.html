Base44 hosting decision: the current Node.js/Express server is not deployed unchanged. During Phase 4, Person A ports its route handlers and services into Base44’s Deno-powered TypeScript backend functions, replaces filesystem JSON with Base44 entities, stores the OpenAI key as a Base44 secret, and deploys the functions to Base44. Canonical system map

The components remain stable; phase impact changes

This is the target component model derived from the initial screenshots and the current code. Today, browser components live together inside widget/widget.js, while backend behavior lives in the Node/Express server/server.js. Phase 1 creates portable service boundaries; Phase 4 moves those backend services into Base44 functions and entities.

What “host inside Base44” means

  • Backend code becomes self-contained TypeScript functions under base44/functions/<name>/entry.ts.
  • Each function uses Deno.serve() and Web-standard Request/Response APIs.
  • Each deployed function receives an endpoint under /functions/<function-name>.
  • OpenAI credentials are Base44 project secrets read with Deno.env.get().
  • Sessions, events, companies, and workflows become Base44 entities—not local JSON files.

What does not move unchanged

  • No Express app, app.listen(), or Express middleware chain.
  • No Node filesystem logging with fs/path.
  • No assumption of a long-running in-memory process or shared maps between invocations.
  • No browser access to service-role permissions or the OpenAI secret.
  • Base44 site hosting remains a static SPA/assets surface; server behavior lives in backend functions.

Verified Base44 platform assumptions

Runtime and deployment facts used by this plan

Backend functions

  • Base44 backend functions are Deno-powered TypeScript functions, use Deno.serve(), and return Web Response objects.
  • Functions live under base44/functions/, can be tested with base44 dev, and each deployed function receives an HTTP endpoint.
  • Base44 currently documents a limit of 50 backend functions per project.
  • Functions can use createClientFromRequest() and service-role entity access after request validation.

Base44 backend functions documentation ↗

Secrets, deployment, and hosting

  • Project secrets are set with base44 secrets set and exposed to functions through Deno environment variables.
  • base44 functions deploy deploys functions; base44 deploy deploys project resources together.
  • Base44 site hosting supports static SPAs/assets; backend execution belongs in functions rather than Node SSR/server components.
  • Direct HTTP function calls do not automatically represent an authenticated Base44 user, so external embeds need explicit client/domain validation.
